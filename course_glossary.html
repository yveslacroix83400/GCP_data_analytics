Cloud Data Analytics Glossary

A comprehensive list of terms and definitions from all five courses.

A

Abstraction: A concept used to understand complex concepts and ideas by focusing on the most essential parts.

Advanced provisioning: A cloud delivery model in which the user signs a formal agreement with the cloud service provider and either pays a set price, or is billed monthly.

Alerting: A feature that enables dashboard users to receive a notification when predetermined conditions are met or exceeded.

Algorithm: A process or set of rules followed for a specific task.

Apache Hadoop: A framework that distributes the processing of data across clusters of computers.

Application programming interface (API): A protocol that enables one application to interact with another application.

Architecture: A framework that defines the design of a technical solution.

Archival policy: An outline of where and how data is stored once an analysis project is complete.

Array: A collection of values of the same data type.

Audit: A formal examination of how users are accessing data in order to ensure safe and appropriate access, while identifying and solving any data concerns.

Automated transformation: The use of processing and scripting tools with less or no programming, compared to manual transformation.

Automatic refreshing: A process that enables dashboards to be automatically updated at regular intervals.

Automation: The use of software, scripting, or machine learning to perform data tasks with less or no manual effort.

B

Batch processing: A type of data processing that takes place at a specific time and requires the data to be in a stable state at the time of the event.

Batch transformation: A process where data is transformed and loaded as a single group, rather than a continuous stream.

BigQuery: A serverless data warehouse on Google Cloud that enables high-speed data queries and analysis.

Boolean: A data type that can only have two values, either true or false.

Bigtable: A cloud-based, NoSQL database service offered by Google Cloud.

C

Cloud computing: The delivery of on-demand computing services, including servers, storage, databases, networking, software, analytics, and intelligence over the internet (“the cloud”) to offer faster innovation, flexible resources, and economies of scale.

Column: A collection of data of a single type.

Column-oriented database: A data-storage system that maintains data in columns, rather than rows, which is better for analytical processing.

Contextual data: Data that provides a full picture of a particular dataset, including who created it, where it came from, and how it was collected.

Cost management: The process of tracking, controlling, and optimizing cloud computing expenses.

D

Data: A collection of facts, statistics, or items of information.

Data analyst: A professional who collects, cleans, analyzes, and interprets data to uncover insights that help organizations make better decisions.

Data architecture: The framework that defines the data elements of an organization and how they are stored, managed, and used.

Data cleaning: The process of detecting and correcting errors or inconsistencies in a dataset to improve its quality.

Data lake: A centralized repository that allows you to store all of your structured and unstructured data at any scale.

Data lakehouse: A modern data architecture that combines the best features of a data lake and a data warehouse, enabling both traditional business intelligence and advanced analytics.

Data management: The practice of collecting, organizing, and maintaining data so that it can be retrieved and used efficiently.

Data model: A visual representation of data and how different data elements relate to each other.

Data pipeline: A set of processes that move and transform data from one system to another.

Data privacy: The right of an individual to have their personal data protected from unauthorized access or use.

Data processing: The process of converting raw data into a more usable and meaningful form.

Data security: The practice of protecting digital information from unauthorized access, corruption, or theft throughout its entire lifecycle.

Data storage: The practice of holding data in a safe and secure place, allowing for easy access and retrieval.

Data transformation: The process of converting data from one format or structure into another.

Data warehouse: A large, central repository that stores data from multiple sources in a structured format for analysis and reporting.

Dataset: A collection of related data.

Dataproc: A fully managed cloud service on Google Cloud for running Apache Hadoop and Spark jobs.

Dataplex: A Google Cloud service that helps you discover, manage, and monitor your data across data lakes, data warehouses, and data marts.

Dimension: A qualitative value used in LookML to categorize or segment data.

Document: A single unit of data in a NoSQL database, often in a flexible format like JSON.

E

ETL (Extract, Transform, Load): A data integration process that combines data from multiple sources, transforms it into a clean and consistent format, and loads it into a target system.

Exploratory data analysis (EDA): An approach to analyzing data sets to summarize their main characteristics, often with visual methods.

F

Fact table: A central table in a data warehouse that contains measures and foreign keys to dimension tables.

Foreign key: A field in one table that uniquely identifies a row of another table.

G

Git: A distributed version control system for tracking changes in source code during software development.

H

Hierarchical data: Data organized in a tree-like structure, with parent-child relationships.

I

Ingestion: The process of moving data from one or more sources to a data destination.

Incremental loading: A method where only the data that has changed since the last load is moved to the target system.

J

JSON (JavaScript Object Notation): A lightweight data-interchange format.

Join: A SQL clause used to combine rows from two or more tables, based on a related column between them.

K

Key: A unique identifier for a row in a table.

L

Lineage: The lifecycle of data, including its origin, where it moves, and its transformations.

Looker: A business intelligence and data analytics platform on Google Cloud.

LookML: A declarative modeling language used in Looker to define data models.

M

Measure: A quantitative value used in LookML to aggregate or calculate data.

Metadata: Data about data.

N

NoSQL: A category of databases that do not use the relational model, suitable for unstructured data.

Normalization: The process of organizing data in a database to reduce data redundancy.

O

Outer join: A join that returns both matched and unmatched rows from one or both tables.

P

Pandas: An open source data analysis and manipulation tool that can be used with Python.

Partitioning: A process for dividing data into separate segments, improving query performance.

Python: A high-level, general-purpose programming language.

R

R: A programming language used for statistical computing.

Relational database: A database that stores data in a structured format using tables, rows, and columns.

Row: A collection of data for a single record.

S

Schema: The logical container for all database objects.

Structured data: Data that is organized and stored in a fixed format, like a table.

Structured Query Language (SQL): A language used to manage and retrieve data from relational databases.

Streaming ingestion: A method of collecting and processing data as soon as it becomes available.

T

Traditional computing: A computing model that uses physical hardware and software on-premises.

Transform: The stage of taking data, cleaning it, and putting it into a standard format in a data pipeline.

U

Unstructured data: Data that is not organized in any easily identifiable way.

Uptime: The amount of time a machine is operational.

V

Validation rules: A set of instructions that specify the standards that must be met for data to be valid, and how to handle data errors.

Versioning: The process of creating a unique way of referring to data.
